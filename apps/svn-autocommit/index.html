Product story

Make SVN maintenance less repetitive.

SVN AutoCommit helps developers and teams keep Subversion working copies updated and committed with less manual effort. It can discover SVN working copies, monitor changes, and commit either today's changes or all changes by choice.

Integration with TortoiseSVN and svn.exe makes it fit existing SVN environments, while tray controls and notifications keep the workflow lightweight.

It is designed for developers who manage several working copies and want cleaner history, fewer missed commits, and faster update routines.

Why users choose it

A tray-based helper for SVN teams.

  • Detect SVN working copies across folders or drives
  • Commit today's changes or all changes based on your workflow
  • Run one-click updates across working copies
  • Work with TortoiseSVN and svn.exe setups
  • Reduce repetitive version-control maintenance

Key features

Built for a smoother Windows workflow

  • 🕒 Commit scope: Today / All — choose exactly what to commit
  • 🔍 Recursive watch (folder/drive) + auto-detect SVN working copies
  • 🧰 TortoiseSVN / svn.exe integration — precise add/remove staging
  • ⚡ Debounced auto-commit + single Start/Stop toggle
  • 🔄 One-click Update across all working copies
  • 🪟 Windows tray app (minimize to tray, bottom-right)
  • Commit Scope switch — choose Today only or All changes per working copy.
  • Single Start/Stop toggle — one button; text & icon update automatically (tray action syncs too).
  • Smarter multi-repo Update/Commit — chunked paths to avoid command-line limits; better externals handling.
  • Precise staging with svn.exe — auto-add/remove only the targeted files; smoother TortoiseProc commits.
  • Midnight rollover safe — “today” window resets automatically at 00:00.
  • Performance tweaks — debounced scanning, adjustable intervals, skips .svn for less I/O.
